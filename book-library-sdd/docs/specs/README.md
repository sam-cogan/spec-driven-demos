# Specs

In-flight and completed feature specs live here, one folder per feature.

Each folder follows the pattern:

- `spec.md` — what & why (produced by `product-analyst`)
- `architecture.md` — how (produced by `solution-architect`)
- `plan.md` — ordered task list (produced by `tech-lead`)
- `validation.md` — proof of satisfaction (produced by `tech-lead`)

See the [demo script](../demo-script.md) for the full flow.
